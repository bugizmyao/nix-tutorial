# Nix言語 — まず特徴を3つ

Nix言語は、パッケージや環境を**定義する**ためのDSL（小さなプログラミング言語）です。

押さえるべき特徴は3つ:

1. **純粋関数型** — 副作用なし、同じ入力なら同じ出力
2. **遅延評価** — 必要になるまで計算しない
3. **動的型付け** — JavaScriptに似た感覚

> [note] 「Nixでスクリプトを書く」のではなく「Nixで設定を記述する」言語です。CやPythonとは目的が違います。

---

# 基本値 — まずはリテラル

```nix
# 数値
42
3.14

# 文字列
"hello"
''
  複数行
  文字列
''

# bool / null
true
false
null

# リスト（カンマなし！スペース区切り）
[ 1 2 3 ]
[ "a" "b" "c" ]
```

> [hint] リストの区切りがカンマでなく **スペース** なのが Nix の特徴。最初は違和感がありますが慣れます。

---

# attrset — 全ての設定の基本

attribute set（略して attrset）は、JSの object に近いキー・値ペアの集合です。

```nix
{
  name = "Bim";
  age = 30;
  skills = [ "Go" "Java" "Nix" ];
}
```

アクセスは `.` で:

```nix
let user = { name = "Bim"; age = 30; };
in user.name      # → "Bim"
```

> [hint] flake.nix も shell.nix も、突き詰めると「巨大な attrset を返している」だけです。

---

# let-in — ローカル変数

「一時的に変数を定義して、`in` のあとで使う」スタイル。

```nix
let
  x = 1;
  y = 2;
  z = x + y;
in
  z * 10        # → 30
```

複数行の式で値を組み立てるときの基本パターン。

---

# with 式 — attrset を「展開」する

`with` を使うと、attrset のフィールドを名前空間なしで使えます。

```nix
let pkgs = { python3 = "p"; nodejs = "n"; jq = "j"; };
in with pkgs; [ python3 nodejs jq ]
# → [ "p" "n" "j" ]
```

shell.nix でよく見る `with pkgs; [ ... ]` がまさにこれ。

> [note] 多用するとどこから来た名前か追えなくなるので、最近は `inherit` を好む人も多いです。

---

# 関数 — 単一引数

矢印みたいに `:` を使います。

```nix
let double = x: x * 2;
in double 21     # → 42
```

呼び出しは **スペース区切り**（カッコ不要）。これも Nix の特徴。

```nix
let add = a: b: a + b;     # カリー化された関数
in add 3 4                  # → 7
```

`a: b: ...` は `a: (b: ...)` の意味で、自動的にカリー化されます。

---

# 関数 — attrset 引数（よく使う）

実用上は、引数を attrset で受け取るパターンが圧倒的に多いです。

```nix
{ name, age }:
  "Hello ${name}, you are ${toString age}"
```

呼び出し:

```nix
greet { name = "Bim"; age = 30; }
# → "Hello Bim, you are 30"
```

これは shell.nix の `{ pkgs ? import <nixpkgs> {} }: ...` と同じ形ですね。

---

# デフォルト値 と `...`

```nix
{ name, greeting ? "Hello", ... }:
  "${greeting}, ${name}"
```

- `?` でデフォルト値
- `...` で「他の引数も受け取るがここでは使わない」

「呼び出し側が余計なキーを渡しても無視する」設定でよく使います。

---

# import — ファイル分割

別のファイルを読み込めます。

```nix
# default.nix
import ./pkgs.nix
```

```nix
# pkgs.nix
{ pkgs ? import <nixpkgs> {} }:
  with pkgs; [ python3 nodejs jq ]
```

```nix
# 使う側
let myPkgs = import ./pkgs.nix {};
in mkShell { buildInputs = myPkgs; }
```

設定が大きくなったら分割すると見通しが良くなります。

---

# rec attrset — 自己参照

通常の attrset は自分自身を参照できませんが、`rec` をつけると参照できます。

```nix
rec {
  base = "Hello";
  greeting = "${base}, world!";
}
# greeting → "Hello, world!"
```

> [hint] `rec` は便利ですが、依存順を読みにくくするので濫用注意。多くの場合 `let` で十分です。
