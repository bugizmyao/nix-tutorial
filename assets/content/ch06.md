# Flakes とは

Flakes は Nix のモダンな入口です。

- **依存をハッシュで固定** — `flake.lock` で完全に再現可能
- **構造化された出力** — devShells / packages / nixosConfigurations / darwinConfigurations / ...
- **`nix` コマンドの第一級市民** — `nix run` / `nix develop` / `nix build` 全部 Flake 前提

旧式の `default.nix` / `shell.nix` は今でも動きますが、新しく作るなら Flake です。

> [note] Determinate Installer ならデフォルトで Flakes が有効。何も設定しなくていいです。

---

# nix flake init — 雛形を作る

空のディレクトリで:

```bash
$ mkdir myflake && cd myflake
$ nix flake init
$ ls
flake.nix
```

中身はこんな感じ:

```nix
{
  description = "A very basic flake";
  inputs.nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

  outputs = { self, nixpkgs }: {
    # ここに出力を書く
  };
}
```

---

# flake.nix の3つの要素

```nix
{
  description = "...";    # 説明文（オプション）
  inputs = { ... };       # 依存する Flake たち
  outputs = ({ self, ... }: { ... });  # 提供するもの
}
```

ポイント:

- `inputs` は **どこから何を持ってくるか**
- `outputs` は **何を提供するか** を返す関数
- `self` は flake自身（再帰参照用）

---

# inputs — 依存を指定

代表的な input:

```nix
inputs = {
  # nixpkgs（必須級）
  nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

  # システム共通化ユーティリティ
  flake-utils.url = "github:numtide/flake-utils";

  # nix-darwin
  nix-darwin = {
    url = "github:LnL7/nix-darwin";
    inputs.nixpkgs.follows = "nixpkgs";  # ← nixpkgsを共通化
  };

  # home-manager
  home-manager = {
    url = "github:nix-community/home-manager";
    inputs.nixpkgs.follows = "nixpkgs";
  };
};
```

`follows` は **「俺の使ってる nixpkgs と同じやつを使え」** という指示。重複ダウンロードを防げます。

---

# outputs — 関数の形

`outputs` は **関数** です。引数は `self` と各 input の名前。

```nix
outputs = { self, nixpkgs, flake-utils, ... }: {
  # ここで提供する各種設定を返す
}
```

`...` で「他にも input があれば受け取る」を明示しておくと、後で input を足したときに楽です。

---

# outputs の主な種類

| キー | 用途 |
|------|------|
| `devShells.<system>.<name>` | 開発環境（`nix develop`） |
| `packages.<system>.<name>` | パッケージ（`nix build`） |
| `apps.<system>.<name>` | 実行可能なアプリ（`nix run`） |
| `nixosConfigurations.<host>` | NixOS のシステム設定 |
| `darwinConfigurations.<host>` | nix-darwin の設定 |
| `homeConfigurations.<user>` | home-manager の設定（standalone） |

`<system>` は `aarch64-darwin`（M1/M2 Mac）/ `x86_64-darwin`（Intel Mac）/ `x86_64-linux` などです。

---

# 最小の devShell flake

```nix
{
  inputs.nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

  outputs = { self, nixpkgs }:
    let
      system = "aarch64-darwin";
      pkgs = import nixpkgs { inherit system; };
    in {
      devShells.${system}.default = pkgs.mkShell {
        buildInputs = with pkgs; [ python3 nodejs_20 jq ];
      };
    };
}
```

使う:

```bash
$ nix develop
[nix-develop]$ python3 --version
```

# nix flake show / check

Flake が何を提供しているか確認:

```bash
$ nix flake show
git+file:///path/to/myflake
└───devShells
    └───aarch64-darwin
        └───default: development environment 'nix-shell'
```

整合性チェック:

```bash
$ nix flake check
```

書き間違いがあると教えてくれる便利コマンドです。

# flake.lock — バージョンの固定

`nix develop` を初めて叩くと、`flake.lock` というファイルが生成されます。

```json
{
  "nodes": {
    "nixpkgs": {
      "locked": {
        "lastModified": 1700000000,
        "rev": "abc123def...",
        "type": "github",
        "owner": "NixOS",
        "repo": "nixpkgs"
      }
    }
  }
}
```

これがあれば、**1年後に同じコマンドを叩いても全く同じ環境** が再現できます。

更新したい時:

```bash
$ nix flake update      # 全input更新
$ nix flake lock --update-input nixpkgs   # nixpkgsだけ更新
```

> [hint] `flake.lock` は **必ずGitにコミット** しましょう。これが再現性の鎖の最後の一手です。

# 演習 — 自分用 flake を作る

> [try] 以下を試してみましょう。
>
> ```bash
> $ mkdir ~/playground/myflake && cd ~/playground/myflake
> $ nix flake init
> ```
>
> 生成された `flake.nix` を編集して devShell を追加:
>
> ```nix
> {
>   inputs.nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
>   outputs = { self, nixpkgs }:
>     let
>       system = "aarch64-darwin";
>       pkgs = import nixpkgs { inherit system; };
>     in {
>       devShells.${system}.default = pkgs.mkShell {
>         buildInputs = with pkgs; [ python3 jq ];
>         shellHook = "echo '🎉 my first flake'";
>       };
>     };
> }
> ```
>
> ```bash
> $ nix develop
> 🎉 my first flake
> [nix-develop]$ python3 --version
> [nix-develop]$ jq --version
> ```

次の章では、これを実際の SES案件 (Spring Boot) に応用していきます。
