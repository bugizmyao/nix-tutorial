# home-manager とは

`home-manager` は、**ユーザーレベルの設定** を Nix で宣言的に管理するツールです。

- `~/.zshrc`, `~/.gitconfig`, `~/.config/nvim/...` などの dotfiles
- ユーザーがインストールするツール（CLI）
- 各ツールの設定（alias / theme / plugins）

nix-darwin が「Mac全体」、home-manager が「ユーザー領域」 ——
**この2つを組み合わせるとMacの全てが宣言的になる** という最終形にたどり着きます。

---

# 2つの導入パターン

| パターン | 特徴 |
|---------|------|
| **standalone** | home-manager 単独で動かす。Mac本体に Nix が入ってさえいればOK |
| **nix-darwin統合** | nix-darwin の中で home-manager を呼ぶ。設定が1箇所にまとまる |

長期運用を考えるなら **nix-darwin統合** がおすすめ。このチュートリアルではこちらを採用します。

---

# home.nix の基本構造

```nix
{ config, pkgs, ... }:
{
  home.username = "bim";
  home.homeDirectory = "/Users/bim";
  home.stateVersion = "23.11";       # 初回決めたら変えない

  home.packages = with pkgs; [
    ripgrep
    fd
    bat
    htop
  ];

  programs.home-manager.enable = true;
}
```

各プロパティの意味:

- `home.packages` — ユーザー領域にインストールするパッケージ（PATHに入る）
- `programs.*` — 各種ツールの宣言的設定（次以降のスライドで詳細）

---

# dotfiles を宣言管理する

`home.file` を使えば、任意のファイルを `$HOME` に配置できます。

```nix
home.file = {
  ".gitignore_global".source = ./files/gitignore_global;
  ".config/karabiner/karabiner.json".source = ./files/karabiner.json;
};
```

- `.source` でファイルパスを指定（リポジトリからコピー）
- `.text` で中身を直書きもできる

> [hint] でも実際は次のスライド以降の `programs.*` を使う方が圧倒的に楽です。

---

# zsh 設定

```nix
programs.zsh = {
  enable = true;
  enableCompletion = true;
  autosuggestion.enable = true;
  syntaxHighlighting.enable = true;

  shellAliases = {
    ll = "ls -la";
    gs = "git status";
    cdh = "cd ~/home";
  };

  history = {
    size = 10000;
    save = 10000;
    share = true;
  };

  initContent = ''
    # カスタム初期化スクリプト
    export EDITOR=nvim
  '';
};
```

これだけで `.zshrc` 相当が宣言的に生成されます。

# starship / direnv

プロンプトと direnv も宣言的に。

```nix
programs.starship = {
  enable = true;
  settings = {
    add_newline = false;
    character = {
      success_symbol = "[➜](bold green)";
      error_symbol = "[➜](bold red)";
    };
  };
};

programs.direnv = {
  enable = true;
  nix-direnv.enable = true;
  enableZshIntegration = true;
};
```

これで Mac新調時にプロンプトと direnv が即座に復活します。

---

# neovim

```nix
programs.neovim = {
  enable = true;
  defaultEditor = true;
  viAlias = true;

  plugins = with pkgs.vimPlugins; [
    nvim-treesitter.withAllGrammars
    telescope-nvim
    nvim-lspconfig
  ];

  extraLuaConfig = ''
    vim.opt.number = true
    vim.opt.relativenumber = true
  '';
};
```

> [note] プラグインも Nix で固定されるので、再現性100%。

# nix-darwin との統合

home-manager を nix-darwin から呼ぶには、`flake.nix` の `inputs` に追加し、`darwinConfigurations` で `imports` に挟みます。

```nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
    nix-darwin.url = "github:LnL7/nix-darwin";
    home-manager.url = "github:nix-community/home-manager";
    home-manager.inputs.nixpkgs.follows = "nixpkgs";
  };

  outputs = { self, nixpkgs, nix-darwin, home-manager }: {
    darwinConfigurations."myMac" = nix-darwin.lib.darwinSystem {
      system = "aarch64-darwin";
      modules = [
        ./configuration.nix
        home-manager.darwinModules.home-manager
        {
          home-manager.useGlobalPkgs = true;
          home-manager.useUserPackages = true;
          home-manager.users.bim = import ./home.nix;
        }
      ];
    };
  };
}
```

これで `darwin-rebuild switch --flake .` 一発で「システム + dotfiles」両方が同時に反映されます。

> [hint] ここまで来たら、Mac新調時は `git clone → darwin-rebuild switch` で完全復元できる世界です。
