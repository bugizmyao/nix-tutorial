# Nixとは何か

Nixはひとことで言うと、**「再現可能な環境」を作るためのパッケージマネージャ**です。

普段使っている Homebrew や asdf と似ているけれど、目指しているゴールがちょっと違います。このチュートリアルでは、Macの開発環境を Nix で宣言的に管理できるところまで一緒に進んでいきます。

> [note] このチュートリアルでは macOS を前提に進めます。Linux でもほぼ同じです。

---

# Nixには3つの顔がある

「Nix」という言葉は、文脈によって3つのものを指します。

| 名前 | 何か |
|------|------|
| **Nix（パッケージマネージャ）** | パッケージのビルド・インストール・管理 |
| **Nix言語** | パッケージや環境を定義するためのDSL |
| **NixOS** | Nixで構築されたLinuxディストロ |

今回の学習対象は **Nix（パッケージマネージャ）と Nix言語** の2つ。NixOS は Linux のみなので扱いません。

代わりに最後に **nix-darwin（macOSのシステム管理）** と **home-manager（ユーザー設定管理）** をやります。

---

# なぜ Nix か？4つの強み

```text
1. 再現性  — 同じ設定なら、いつ・どこでも同じ環境
2. 隔離性  — プロジェクトごとに環境を完全分離できる
3. 宣言的  — 「何が欲しいか」を書けば、Nixが整える
4. ロールバック — 過去の世代に一発で戻せる
```

例えば、3年前のプロジェクトを開いたとき、`flake.nix` があれば Node.js のバージョンも依存ツールも当時のまま再現できます。

> [hint] 「あのプロジェクト、Node 14 だっけ 16 だっけ？」がなくなる世界です。

# Homebrew / asdf との違い

| 特徴 | Homebrew | asdf | Nix |
|------|:--------:|:----:|:---:|
| プロジェクト単位の隔離 | ✗ | △（言語別） | ◎ |
| 宣言的な定義ファイル | ✗ | △（.tool-versions） | ◎ |
| ロールバック | ✗ | ✗ | ◎ |
| システム設定の管理 | ✗ | ✗ | ◎ (nix-darwin) |
| dotfiles 管理 | ✗ | ✗ | ◎ (home-manager) |
| 学習コスト | 低 | 低 | 中〜高 |

Homebrew は「Mac全体に1つの環境」、Nix は「プロジェクトごとに独立した環境 + システム全体も宣言的」というイメージです。

> [note] Homebrew をやめる必要はありません。Nix と併用するのが現実的です（nix-darwin から Homebrew を呼ぶこともできます）。
