# 目標 — 実案件に応用する

ここまで覚えたことを使って、**実際の SES案件の開発環境** を flake.nix に落とし込みます。

想定: Spring Boot 案件
- JDK 21
- Maven
- 補助ツール（jq / git / direnv連携）

ゴール: チームの新メンバーが `git clone` → `direnv allow` だけで開発開始できる状態。

---

# 必要なツールを洗い出す

```text
言語ランタイム:  JDK 21
ビルドツール:   Maven
DB:            PostgreSQL（ローカル開発用）
補助:          jq, ripgrep, direnv
```

これらを `flake.nix` の `buildInputs` に書けばOK。

> [note] Spring Boot 用の Gradle が必要なら `gradle` も。今回は Maven 想定。

---

# flake.nix 雛形

```nix
{
  description = "SES案件 dev environment";

  inputs.nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

  outputs = { self, nixpkgs }:
    let
      system = "aarch64-darwin";
      pkgs = import nixpkgs { inherit system; };
    in {
      devShells.${system}.default = pkgs.mkShell {
        buildInputs = with pkgs; [
          jdk21
          maven
          postgresql_16
          jq
          ripgrep
        ];

        shellHook = ''
          echo "☕ Spring Boot dev env loaded"
          echo "  java $(java --version | head -n1)"
          echo "  mvn  $(mvn --version | head -n1)"
        '';
      };
    };
}
```

---

# 使ってみる

```bash
$ cd ~/projects/your-spring-app
$ nix develop
☕ Spring Boot dev env loaded
  java openjdk 21.0.x ...
  mvn  Apache Maven 3.9.x ...
```

```bash
[nix-develop]$ mvn spring-boot:run
```

JDK / Maven は Mac本体には何も入れていないのに、subshellの中だけで完璧に動きます。

> [hint] チームに JDK バージョンを揃えてもらう交渉が要らなくなります。

---

# flake-utils で複数system対応

`aarch64-darwin` だけにハードコードすると、IntelMacのメンバーが使えません。`flake-utils` でクロスplatform化:

```nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
    flake-utils.url = "github:numtide/flake-utils";
  };

  outputs = { self, nixpkgs, flake-utils }:
    flake-utils.lib.eachDefaultSystem (system:
      let pkgs = import nixpkgs { inherit system; };
      in {
        devShells.default = pkgs.mkShell {
          buildInputs = with pkgs; [ jdk21 maven ];
        };
      }
    );
}
```

これで Apple Silicon / Intel / Linux すべてで動きます。

# 環境変数を設定する

`shellHook` で env変数を仕込めます。

```nix
shellHook = ''
  export JAVA_OPTS="-Xmx2g"
  export SPRING_PROFILES_ACTIVE=local
  export DATABASE_URL=postgresql://localhost:5432/myapp
  echo "✅ env vars exported"
'';
```

Mac本体の `.zshrc` を汚さずに、プロジェクトごとに env を持てます。

---

# direnv で自動切り替え

毎回 `nix develop` を叩くのは面倒。`.envrc` を置いて direnv に任せます。

```bash
$ echo "use flake" > .envrc
$ direnv allow
```

これで `cd ~/projects/your-spring-app` した瞬間に自動で env がロードされ、JDK と Maven が使える状態になります。

> [hint] direnv + Flake は最強コンボ。一度味わうと戻れません。

---

# チームに展開する

リポジトリにこのファイルをコミット:

```text
your-spring-app/
├── flake.nix      # 環境定義
├── flake.lock     # バージョン固定
├── .envrc         # direnv 設定
└── src/...
```

新メンバーの手順:

```bash
$ git clone ...
$ direnv allow
# → 自動で JDK 21 + Maven がロードされる
```

> [note] 新メンバーは事前に Nix と direnv だけインストールしておけばOK。READMEに書きましょう。

# 演習 — 自分の案件で書いてみる

> [try] 自分の SES案件のディレクトリで `flake.nix` を書き、`nix develop` で入れることを確認しましょう。
>
> 必要なツールを洗い出して `buildInputs` に並べ、`shellHook` でバージョン表示するところまで。
>
> ```bash
> $ cd ~/projects/your-project
> $ vim flake.nix       # 上の雛形を参考に
> $ nix develop
> ```
>
> うまく動いたら、`.envrc` を作って direnv 連携も試してみてください。
